# 31 — Soft Launch Policy

## Purpose

Agency Terminal should not immediately become authoritative. The ledger must earn trust before it affects rank, access, or clearance decisions.

## Launch Principle

```text
Collect first. Calibrate second. Influence authority last.
```

## Shadow Mode

Recommended first phase:

```text
Duration: 2–4 weeks
Profiles: officer-only
Scores: calculated but not authority-bearing
Clearance: not automatically affected by score
Role changes: manual only
Evidence: real submissions accepted
Reviews: real reviewer actions recorded
```

## What Shadow Mode Tests

```text
Are metric point values reasonable?
Are reviewers overloaded?
Are evidence categories clear?
Are group-credit flows practical?
Are intel records too sensitive?
Are appeals needed often?
Are doctrine challenges useful or noisy?
Does the bot reduce officer workload?
```

## Exit Criteria

Do not leave shadow mode until:

```text
no critical schema changes pending
reviewers understand quorum process
stale escalation works
appeals process exists
score reversal process exists
public/officer profile split confirmed
retention/export policy accepted
The Agency leadership accepts the social contract
```

## Phase Progression

### Phase 0 — Dry Run

```text
Test server or private channels.
Fake evidence.
No real scoring.
```

### Phase 1 — Shadow Ledger

```text
Real evidence.
Real reviews.
Officer-only profiles.
No authority impact.
```

### Phase 2 — Advisory Ledger

```text
Profiles visible to members where safe.
Scores may inform discussion.
No automatic promotions.
```

### Phase 3 — Operational Ledger

```text
Leadership may reference ledger in clearance/rank decisions.
Bot still does not auto-promote or auto-demote.
```

## Failure Criteria

Pause rollout if:

```text
evidence queues stall
intel leaks occur
reviewers complain of overload
score disputes dominate usage
contract info leaks
bot permission drift causes private ticket exposure
```

## Acceptance Criteria

- Shadow mode is documented before deployment.
- Profiles default officer-only during soft launch.
- Leadership understands score is advisory.
- Exit criteria are explicit.
