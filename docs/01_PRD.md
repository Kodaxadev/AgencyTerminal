# Agency Terminal PRD

## Product summary

Agency Terminal is a Discord-native operations terminal for The Agency // Lux Letifera. It turns enlistment, mercenary contracts, intelligence reports, performance evidence, doctrine challenges, and clearance requests into structured, auditable workflows.

The product is not a generic ticket bot. It is a contribution ledger with ticket-shaped intake.

## Doctrine alignment

The Agency doctrine creates the product's foundation:

- Favoritism is treated as a liability.
- Subjectivity is treated as noise.
- Clearance, authority, and access are determined by measurable performance.
- Agents are expected to submit critique and analysis.
- Free agency is allowed if Agency interests and diplomatic accords remain uncompromised.

Agency Terminal operationalizes this by making contributions legible, reviewable, credited, and reversible only through strict procedure.

## Goals

1. Create structured intake for Agency workflows.
2. Make contribution evidence visible to reviewers.
3. Convert validated evidence into immutable score events.
4. Preserve auditability for approvals, denials, reversals, and overrides.
5. Provide a Discord-native UX that matches Agency tone and terminal language.
6. Keep v1 human-in-the-loop: no auto-promotion, auto-denial, or automatic rank mutation.

## Non-goals for v1

- No automatic role grants.
- No automatic demotions.
- No direct on-chain writes.
- No multi-tribe SaaS billing.
- No fully automated killboard validation.
- No Signal Vault write dependency.
- No FrontierWarden dependency.

## Core workflows

1. Enlistment Protocol
2. Contract Intake
3. Intel Report
4. Performance Evidence
5. Doctrine Challenge
6. Clearance Request
7. General Operational Ticket

## Product principle

The bot should recommend, route, record, and expose structured evidence. It should not decide political value or replace leadership judgment.
